# Глоссарий

> **Для кого:** оператор, разработчик  
> **Время чтения:** ~5 мин  
> **Что узнаете:** термины протокола TDS и Queryeasy, которые встречаются в конфигурации и логах.

## Протокол и сообщения

| Термин | Определение |
| --- | --- |
| **TDS** (Tabular Data Stream) | Бинарный протокол Microsoft SQL Server для обмена данными между клиентом и сервером. Queryeasy работает на этом уровне, не на уровне ODBC/JDBC. |
| **TDS packet** | Единица передачи с заголовком (тип, длина, status). Одно логическое сообщение может состоять из нескольких пакетов. |
| **SQL Batch** | TDS-сообщение с текстом SQL-запроса в Unicode. Queryeasy может декодировать его и применять rewrite. |
| **RPC Request** | TDS-сообщение вызова хранимой процедуры. Rewrite поддерживается только для **`sp_executesql`**. |
| **`sp_executesql`** | Системная процедура SQL Server для выполнения динамического SQL с параметрами. ORM (в т.ч. 1С) часто отправляют запросы именно через неё. |
| **`@stmt`** | SQL-текст внутри RPC `sp_executesql`. |
| **`@params`** | Строка объявления параметров `sp_executesql`, например `@P1 datetime2(3), @P2 int`. |

## PreLogin и шифрование

| Термин | Определение |
| --- | --- |
| **PreLogin** | Первый этап TDS-handshake до login. В нём согласуются опции, включая шифрование. |
| **ENCRYPTION** | Опция PreLogin: клиент и сервер договариваются, будет ли использоваться TLS поверх TDS. |
| **Plaintext TDS** | Трафик без TLS. Нужен для decode SQL и rewrite в Queryeasy. |
| **Raw TLS** | Если после PreLogin поток становится TLS, прокси видит зашифрованные байты и не может прочитать SQL. |

## Режимы и поведение Queryeasy

| Термин | Определение |
| --- | --- |
| **ForwardOnly** | Минимальный overhead: multi-packet message пересылается без decode и rewrite. |
| **InspectOnly** | Decode SQL Batch и RPC для логов; на сервер уходит исходный трафик. |
| **DryRun** | Правила rewrite проверяются и логируются, но на сервер отправляется исходный SQL. |
| **Rewrite** | Совпавшие правила реально изменяют SQL Batch или `sp_executesql` перед отправкой. |
| **InspectionCapabilities** | Набор флагов, вычисляемых при старте: что парсить и что переписывать (зависит от `Mode`, `LogSqlText` и `Scope` правил). |
| **FailOpen** | При ошибке rewrite исходный SQL отправляется на сервер; ошибка только в логе. |
| **FailClosed** | При ошибке rewrite сессия завершается; исходный SQL не отправляется. |
| **Session id** | Короткий идентификатор сессии (8 символов), например `[a1b2c3d4]`, во всех строках лога одного подключения. |

## SQL Server и сценарий 1С

| Термин | Определение |
| --- | --- |
| **`datetime2(n)`** | Тип SQL Server с точностью до n знаков дробной части секунды (scale). `datetime2(3)` — миллисекунды, `datetime2(0)` — без дробной части. |
| **`@P1`, `@P2`, …** | Имена параметров в типичных запросах 1С через `sp_executesql`. |
| **Scope** | Область применения правила rewrite: `Any`, `SqlBatch`, `RpcSpExecuteSql`. |
