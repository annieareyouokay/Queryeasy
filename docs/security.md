# Безопасность

> **Для кого:** оператор  
> **Время чтения:** ~4 мин  
> **Что узнаете:** риски логирования и границы ответственности прокси.

## Чувствительные данные в логах

Queryeasy может выводить в консоль:

- полный **SQL-текст** запросов;
- **значения параметров** `sp_executesql`;
- **hex-preview** TDS payload.

Это включается при `LogSqlText: true` и достаточно подробном `LogLevel` (`Debug` / `Trace`).

SQL-запросы могут содержать персональные данные, токены, пароли в параметрах, фрагменты бизнес-данных и другую чувствительную информацию.

**Не запускайте** прокси с подробным логированием там, где такие данные нельзя сохранять в консоль, терминальные логи или системы сбора логов (ELK, CloudWatch и т.п.).

### Рекомендации

- Production: `LogLevel: "Warn"` или `"Info"`, `LogSqlText: false`, `LogPayloadPreview: false` — как в [appsettings.Production.json](../Queryeasy.Proxy/appsettings.Production.json).
- Включайте детальные логи **временно** на изолированной среде для отладки конкретной проблемы.
- Ограничивайте доступ к машине, где запущен прокси с verbose-логами.

## Queryeasy — не security gateway

Прокси **не** предоставляет:

- аутентификацию и авторизацию клиентов;
- TLS-терминацию и управление сертификатами;
- фильтрацию SQL по политикам безопасности;
- защиту от SQL injection.

Это **диагностический инструмент** для разработки и контролируемых сред. Не выставляйте `ListenHost` на публичные интерфейсы без понимания рисков: любой, кто достучится до порта прокси, может пытаться подключиться к SQL Server через него (с учётными данными клиента).

### Рекомендации для развёртывания

- По умолчанию слушать **`127.0.0.1`**, не `0.0.0.0`, если удалённый доступ не нужен.
- Использовать firewall для ограничения доступа к `ListenPort`.
- Не полагаться на прокси как на единственный контроль доступа к базе.

## Rewrite и целостность данных

Режим **`Rewrite`** изменяет SQL и параметры **до** SQL Server. Ошибка в правиле может привести к неверным запросам или неожиданным результатам приложения.

- Сначала **`DryRun`** и проверка логов.
- Для критичных сценариев: `RewriteFailureBehavior: "FailClosed"` после стабилизации правил.
- Мониторьте метрики `rewrite_failed`, `encode_failed` в summary.
